MyTonWallet API

Self-hosted REST API implementing the full MyTonWallet v4.10.7 feature set. Wallets, tokens, staking, swaps, multi-send, and real-time webhook notifications across 11 chains — all via HTTP.

TON · Jettons EVM × 8 chains Solana · SPL TRON · TRC-20 WebSocket real-time SQLite + Cache Proxy Support

Quick Start

Up and running in under a minute.

1

Clone and install

git clone https://github.com/llogzz/MyTonWallet-api
cd MyTonWallet-api
npm install
2

Edit .env if needed — all defaults work out of the box

# Optional: set your Toncenter API key for higher rate limits
TONCENTER_API_KEY=your_key_here
3

Start the server

npm run dev
4

Import your first wallet

curl -X POST http://localhost:3000/wallets/import \
  -H 'Content-Type: application/json' \
  -d '{"mnemonic": "word1 word2 ... word24", "chain": "ton"}'

Proxy Support

Add proxies to proxy.txt (auto-created on startup). Each WebSocket connection picks a random proxy. Failed proxies are retried after 5 minutes. Changes apply instantly — no restart needed.

# Plain host:port
34.44.49.215:80

# Authenticated: host:port:user:pass
34.44.49.215:80:myuser:mypassword

# URL with credentials
myuser:mypassword@34.44.49.215:80

# With explicit scheme
socks5://myuser:mypassword@34.44.49.215:80
http://myuser:mypassword@34.44.49.215:80
Tip: All formats are accepted and auto-detected. Lines starting with # are ignored.

Telegram Notifications

Receive a Telegram message for every new transaction detected across your monitored wallets.

Setup:
  1. Message @BotFather in Telegram → /newbot → copy the token
  2. Find your chat ID via @userinfobot or @Get_myidrobot
  3. Set both variables in .env and restart
.env 
TELEGRAM_BOT_TOKEN=7123456789:AAEm...
TELEGRAM_CHAT_ID=123456789

Example message sent to Telegram on incoming transaction:

📥 5 TON from UQBvW8Y0...
UQDq5UUyPH...

If neither variable is set, Telegram notifications are silently disabled.
