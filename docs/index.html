MyTonWallet API

Self-hosted REST API for multi-chain self-custody wallets. Send, receive, stake, and swap across 11 chains — all via HTTP. Wallets are identified by numeric ID; accounts by address + optional ?chain=.

TON · Jettons EVM × 8 chains Solana · SPL TRON · TRC-20 WebSocket real-time SQLite + Cache RPC Fallback

Routing model

/wallets/:id — wallet management (numeric ID, e.g. /wallets/1)
/accounts/:address — per-address operations with optional ?chain= and ?network= params
One wallet → many accounts across different chains (same BIP39 seed, different derivation paths).

Quick Start

Up and running in under a minute.

1

Clone and install

git clone https://github.com/llogzz/MyTonWallet-api
cd MyTonWallet-api
npm install
2

Copy config — all defaults work out of the box

cp .env.example .env
# Optionally set TONCENTER_API_KEY for higher rate limits
3

Start the server

npm run dev   # http://localhost:3000
4

Generate or import a wallet

# Generate — derives addresses on all chains automatically
curl -X POST http://localhost:3000/wallets/generate \
  -H 'Content-Type: application/json' \
  -d '{"label": "main"}'

# Import existing mnemonic
curl -X POST http://localhost:3000/wallets/import \
  -H 'Content-Type: application/json' \
  -d '{"mnemonic": "word1 word2 ... word24", "chains": ["ton","ethereum","solana"]}'

Proxy Support

Add proxies to proxy.txt (auto-created on startup). Each WebSocket connection picks a random proxy. Changes apply instantly — no restart needed.

# Plain host:port
34.44.49.215:80

# With credentials
34.44.49.215:80:myuser:mypassword

# URL formats
socks5://myuser:mypassword@34.44.49.215:1080
http://myuser:mypassword@34.44.49.215:80

Wallet Management

Wallets are identified by numeric ID. One wallet (one BIP39 seed) → many accounts across chains. All addresses are derived deterministically from the same mnemonic.

POST /wallets/generate Generate a new BIP39 wallet ▶

Generates a 24-word BIP39 mnemonic and derives addresses on all specified chains. The seed is stored encrypted in SQLite.

Body (all optional)

FieldTypeDescription
chainsstring[]Chains to derive. Default: all supported chains
labelstringHuman-readable wallet name
versionstringTON wallet version: W5 (default), v4R2, v3R2
networkstringmainnet (default) or testnet
Request
Response 201

DELETE /wallets/:id/accounts/:address Remove an account ▶

Without ?chain=: removes all chains sharing this address (e.g. all EVM entries for the same 0x address). With ?chain=: removes only that specific entry.

One chain
All chains for address
Response 200
